Count number of binary strings without consecutive 1's in N bit

Problem Link: https://practice.geeksforgeeks.org/problems/consecutive-1s-not-allowed1912/1

Solution
Approach 1 
class Solution{
public:
    vector<vector<long long>>dp;
	long long countStrings(int n) {
	    dp.assign(n+1,vector<long long>(2,-1));
	    return solve(n,0);
	}
	
	long long solve(int n,int prv){
	    if(n==0) return 1;
	    if(dp[n][prv]!=-1) return dp[n][prv];
	    if(prv==1) return dp[n][prv]=solve(n-1,0)%m;
	    else return dp[n][prv]=(solve(n-1,1)%m+solve(n-1,0)%m)%m;
	}
};

Approach 2 
int countStrings(int n)
{
    // a[i] be the number of binary strings of length i which do not contain any two consecutive 1’s and which end in 0
    // b[i] be the number of such strings which end in 1
    int a[n], b[n];
    a[0] = b[0] = 1;
    for (int i = 1; i < n; i++)
    {
        a[i] = a[i-1] + b[i-1];           // We can append either 0 or 1 to a string ending in 0
        b[i] = a[i-1];                    // we can only append 0 to a string ending in 1
    }
    return a[n-1] + b[n-1];
}

vector<vector<vector<int>>>dp;
int numberOfConsecutiveOnes(int N){
    dp.assign(N+1,vector<vector<int>>(2,vector<int>(2,-1)));
    return solve(N,0,0);
}

int solve(int n,int conOne,int prv){
    if(n==0){
        if(conOne==1) return 1;
        return 0;
    } 
    
    if(dp[n][conOne][prv]!=-1) return dp[n][conOne][prv];
    
    if(conOne){
        return dp[n][conOne][prv]=solve(n-1,1,0)+solve(n-1,1,1);
    }else{
        if(prv==1) return dp[n][conOne][prv]=solve(n-1,1,1)+solve(n-1,0,0);
        else return dp[n][conOne][prv]=solve(n-1,0,0)+solve(n-1,0,1);
    }
}

Approach 2

Count of all possible binary strings of length n is 2^n. Compute the number of binary strings without consecutive 1's using the approach discussed above. Let this count be c. Total binary strings with consecutive 1 is 2^n - c.

Number of the integers in the range [0, n] whose binary representations do not contain consecutive 1's

Problem Link: https://leetcode.com/problems/non-negative-integers-without-consecutive-ones/

Idea
  • We start scanning the given number num from its MSB.
  • For every 1 encountered at the ith bit position(counting from 0 from LSB), we add a factor of f[i] to the resultant count.
  • For every 0 encountered, we don't add any factor.
  • We also keep a track of the last bit checked.
  • If we happen to find two consecutive 1's at any time, we add the factors for the positions of both the 1's and stop the traversal immediately.
  • If we don't find any two consecutive 1's, we proceed till reaching the LSB and add an extra 1 to account for the given number num as well, since the procedure considers numbers upto num without including itself.
Explanation

f[i] denote no. of binary string of length i.

10101101 (173) can be written as 10000000 (128 (2^7)) + 100000(32(2^5)) + 1000(8(2^3)) + 100(4(2^2)) + 1(1(2^0)).

In 1 0 0 0 0 0 0 0 (2^7)
   8 7 6 5 4 3 2 1                  If we consider binary string of length of 8 then our no. will exceed the given no. 
   7 6 5 4 3 2 1 0                  If we consider legth of 7 then we will have 00000000 - 0xxxxxxx length (number got counted from 0 to 2^7-1)

Now for extra number's above(>=) 2^7 (1xxxxxxx) next we have 2^5 plus in it .
Adding contribution due to 2^5 
Again if we consider 5 legth we will exceed our no. (will have extra solution) so we will count 100xxxxx (from 2^7 to 2^7 + (2^5-1) get covered).

Similarly we will add the contribution due to 2^3 in this we will count 100100xxx

Next we will count contribution due to 2^2 in this case we will count 10010010xx

Now our counting will begin from 1001001100 since we got two consecutive one in this and smaller value will not effect these two consecutive one so all values after this will have two consecutive two which will be not counted as it will have two consecutive one.

  • Conclusion - First we calculated from 1 to 2^7-1 then from 2^7 to 2^7 + (2^5-1) ... after calculation till 2^7+2^5+2^3+2^2-1 all no. after it will have two consecutive 11 so we will ignore all these. If we don't have the consecutive one then we will have our answer till num-1 and we will add 1 in final.

Explanation of other

Link : https://leetcode.com/problems/non-negative-integers-without-consecutive-ones/discuss/1363063/C%2B%2BPython-DP-on-32-Bits-Clear-explanation-Clean-and-Concise-O(1)

  • Consider the example n = 21 binary representation as 10101, we have following valid combinations (where x can be 0 or 1) which less than or equal to n:
  ✔️ Group 1: 0xxxx, there are dp[4] = 8 combinations
  ✔️ Group 2: 100xx, there are dp[2] = 3 combinations.
  ✔️ Group 3: 10100, there is dp[0] = 1 combination.
  ✔️ Group 4: 10101 (value n ifself), there is 1 combination.
  So there are total 8 + 3 + 1 + 1 = 13 combinations.
  • Let consider the example n = 58, binary representation as 111010, we have following valid combination (where x can be 0 or 1) which less than or equal to n:
  ✔️ Group 1: 0xxxxx, there are dp[5] = 13 combinations.
  ✔️ Group 2: 10xxxx, there are dp[4] = 8 combinations.
  ❌ Group 3: 110xxx, invalid because it contains consecutive ones
  ❌ Group 4: 11100x, invalid because it contains consecutive ones
  So there are total 13 + 8 = 21 combinations.
  We can see that if we met consecutive ones in our binary representation, the later groups will always invalid, so we just return result immediately, don't calculate anymore.

class Solution {
public:
    int findIntegers(int n) {
        int fib[32];
        fib[0]=1;                             
        fib[1]=2;                             
        for(int i=2;i<32;++i){
            fib[i]=fib[i-1]+fib[i-2];
        }
        int ans=0;
        int prev=0;
        for(int i=31;i>=0;--i){
            if((1<<i)&n){
                ans+=fib[i];
                if(prev){                    
                    ans-=1;
                    break;
                }
                prev=1;
            }else{
                prev=0;
            }
        }
        return ans+1;
    }
};
